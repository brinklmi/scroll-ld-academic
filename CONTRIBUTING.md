## ğŸ”§ Technical Contribution Process

### 1. Fork & Clone
```bash
git clone https://github.com/brinklmi/scroll-ld-academic.git
cd scroll-ld-academic
```

### 2. Create a Branch
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

### 3. Make Changes
- Follow PEP 8 style guidelines
- Add tests for new features
- Update documentation as needed
- Ensure all tests pass locally

### 4. Run Tests
```bash
pytest tests/ -v
python scripts/validate_scroll_structure.py
python scripts/check_glyph_integrity.py
```

### 5. Commit with Intention
```bash
git commit -m "feat: Add consciousness field coherence validation"
# or
git commit -m "fix: Correct glyph encoding in decompressor"
```

Use conventional commits: `feat:`, `fix:`, `docs:`, `test:`, `refactor:`

### 6. Push & Create Merge Request
```bash
git push origin feature/your-feature-name
```

Then create a merge request on GitLab with:
- Clear description of changes
- Reference to any related issues
- Test results
- Documentation updates
